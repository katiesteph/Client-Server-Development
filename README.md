# Client-Server-Development
snhu client/server development

This course taught me that maintainable code is really about planning ahead and making life easier for your future self. Creating the CRUD module in Project One paid off in Project Two because I could reuse the same database functions throughout the dashboard instead of rewriting code. It kept everything cleaner, easier to debug, and flexible enough to reuse in future projects that need database access.

Working on the Grazioso Salvare dashboard also felt more like solving a real-world problem than completing a typical assignment. Instead of focusing on one feature at a time, I had to think about what the client actually needed and how different pieces would work together. Breaking the project into smaller steps and testing along the way helped me stay organized. Projects like this show how computer scientists help organizations turn raw data into useful tools, allowing companies like Grazioso Salvare to spend less time managing information and more time using it to make decisions.
